DesktopST 使用者教學

API Key 是什麼？新手申請與省錢測試指南

這份文件給第一次使用 AI 桌面寵物的人。你可以把 API Key 想成「給 AI 服務看的通行證」。有了它，DesktopST 才能幫你的角色連到 OpenAI、Gemini、Grok 或 Claude。

先懂觀念 選哪一家 申請步驟 Token 用量 安全與省錢
把 Key 放進設定，我才能開始聊天。

1. 先把 API Key 想成一張小門票

API 是什麼？

API 就像一個服務窗口。DesktopST 會把你說的話送到 AI 公司的服務窗口，AI 回答後，DesktopST 再把答案顯示給你。

API Key 是什麼？

API Key 是一串很長的密碼。AI 公司看到這串密碼，就知道「這次使用是誰的帳號」，也知道要把用量算在哪個帳號上。

最重要的一句話：API Key 要像提款卡密碼一樣保管。不要貼到公開網站、Discord、GitHub、截圖或聊天訊息裡。

在 DesktopST 裡，API Key 會由你的電腦保存。你仍然要記得：只在官方網站申請，只貼進你信任的程式，不要交給陌生人。

2. 四家服務快速比較

服務 適合誰 新手免費或測試方式 注意事項
OpenAI 想要穩定、通用、中文英文都常用的人。 依帳號 Billing 頁面為準；若有 free credits 會先扣免費額度。許多帳號需要先購買最低預付點數才能繼續使用。 建議設定使用上限，並用便宜模型先測試。
Gemini 想先免費測試、希望用 Google 帳號快速開始的人。 官方有 Free tier，符合地區的使用者可在限制內免費用部分模型。 免費層級有 RPM、TPM、RPD 等限制，超過會出現限速錯誤。
Grok / xAI 想使用 Grok 模型，或想測試 OpenAI 相容介面的人。 以 xAI Console 顯示的 free/promotional credits 或 promo code 為準；沒有可用點數時通常要先買 prepaid credits。 月結帳單預設通常不開，預付點數用完會停止回應，比較容易控制成本。
Claude 重視長文、角色語氣、寫作與整理能力的人。 Claude API 一般採預付 usage credits；本教學不把它列入免費測試。 先買點數才能使用 API，購買點數通常不可退款且有期限。

資訊更新日：2026-05-12。各家方案可能變動，最終仍以官方 Console 與 Billing 頁面顯示為準。

3. 申請 API Key 前，先準備這三樣

電子信箱

用來註冊帳號、收驗證信、收到付款或用量提醒。

手機或驗證方式

有些服務會要求額外驗證，這是為了防止濫用。

付款方式

Claude、xAI 或部分 OpenAI 帳號可能需要先加值。Gemini 免費層級不一定需要先綁卡。

小朋友版理解：帳號是你的名字，API Key 是你的門票，Billing 是你的零用錢紀錄，Token 是每次聊天用掉的小積木。
OpenAI

4. OpenAI API Key 申請方式

  1. 打開 OpenAI Platform，登入或建立帳號。
  2. 進入 Dashboard 後，找到 API Keys 或專案設定裡的 Keys 頁面。
  3. 按下建立新 Key，取一個容易辨認的名字，例如 DesktopST-test。
  4. 把出現的 Key 複製下來，貼進 DesktopST 的 OpenAI API Key 欄位。這串 Key 通常只會完整顯示一次。
  5. 到 Billing 或 Usage 頁面確認是否有免費額度、已購買點數、用量上限與付款狀態。
新手測試建議：先問一句很短的話，例如「請用一句話自我介紹」。確認成功後，再開始長篇角色對話。

OpenAI 官方文件提醒 API Key 是秘密，不應分享或放在前端程式碼中；API 請求會用 Authorization: Bearer 的方式帶上 Key。

Gemini

5. Gemini API Key 申請方式

  1. 打開 Google AI Studio API Keys，登入 Google 帳號。
  2. 如果系統要求接受條款，先看完並接受。新使用者可能會自動建立預設 Project 與 API Key。
  3. 在 API Keys 頁面按下建立 Key，選擇或建立一個 Google Cloud Project。
  4. 複製 Key，貼進 DesktopST 的 Gemini API Key 欄位。
  5. 到 AI Studio 的 Rate limits 或 Billing 頁面確認你的免費限制與是否已升級付費。
Gemini 免費層級常見限制包含每分鐘要求數 RPM、每分鐘輸入 Token 數 TPM、每日要求數 RPD。限制是算在 Project 上，不是每一把 Key 各自重新算。

Google 官方也提醒：初期測試可以暫時直接填 Key，但正式使用要把 Key 保管好；不要放在公開網站或手機 App 前端裡。

Grok / xAI

6. Grok / xAI API Key 申請方式

  1. 打開 xAI Console，登入或建立 xAI 帳號。
  2. 確認目前選到的 Team 是你要使用的團隊。xAI 的 Key 與付款通常是綁在 Team 上。
  3. 進入 API Keys 頁面，建立一把新 Key，名字可填 DesktopST-test。
  4. 複製 Key，貼進 DesktopST 的 Grok API Key 欄位。
  5. 到 Billing 或 API spend management 查看是否有 free/promotional credits、promo code 或已購買的 prepaid credits。
xAI 文件說明 API 消耗會先扣 free/promotional credits，再扣 prepaid credits。若沒有可用點數，通常需要先購買預付點數。

想看花了多少，可以到 xAI Usage Explorer 依日期、模型或 API Key 查看成本與 Token 用量。

Claude

7. Claude API Key 申請方式

  1. 打開 Anthropic Console，登入或建立帳號。
  2. 進入 Console 後，可以先在 Workbench 測試提示詞，熟悉 Claude 的回答方式。
  3. 到 API Keys 或 Account Settings 建立新 Key，名字可填 DesktopST-test。
  4. 複製 Key，貼進 DesktopST 的 Claude API Key 欄位。
  5. 到 Billing 頁面購買 usage credits。沒有點數時，Claude API 通常無法呼叫。
Claude API 的 usage credits 需要先購買；官方說點數會用於 API、Workbench 等用量，購買點數通常不可退款，且會過期。

8. 如何在免費額度或小額預算內測試

  • 先從 Gemini 開始。如果你所在地區符合 Free tier，Gemini 是最容易零成本測試的選項。
  • 先問短句。例如「你好，請用一句話介紹自己」。不要一開始就丟超長角色設定和長篇劇情。
  • 先用便宜模型。每家模型價格不同，先選輕量或 flash/mini 類模型做連線測試。
  • 關掉自動續費或設低上限。還不熟時，不要開太高的自動加值或月結上限。
  • 每天看一次用量。測試期到官方 Console 的 Usage 或 Billing 頁看花費，養成習慣。
免費不等於無限制。免費額度用完、超過每日限制、或請求太頻繁時，AI 可能會回傳錯誤。這不是 DesktopST 壞掉，而是服務端提醒你「今天先休息一下」。

9. Token 是什麼？為什麼會影響費用？

Token 是 AI 用來計算文字的小單位。你可以把它想成「文字積木」。你送出去的文字會變成一堆積木，AI 回答的文字也會變成一堆積木。服務商通常依照輸入 Token 和輸出 Token 計費。

輸入 Token

你送給 AI 的內容，例如角色設定、世界觀、過去聊天紀錄、這次打的句子。

輸出 Token

AI 回答你的內容。回答越長，輸出 Token 越多，費用通常越高。

情境 Token 用量會怎樣 省錢做法
角色設定很長 每次對話都可能帶入更多輸入 Token。 第一版角色卡先簡短，確認好玩後再慢慢加細節。
聊天紀錄保留很多 AI 需要讀更多舊訊息，輸入 Token 增加。 設定只保留最近 N 則，或使用摘要功能。
要求長篇小說式回答 輸出 Token 增加，費用也增加。 測試時先要求「100 字內」或「一句話」。
群組多角色互相說話 每個角色都可能產生一次 API 呼叫。 先限制回合數，例如每次最多 1 到 3 輪。

10. 安全與省錢檢查表

Key 安全

  • 不要把 API Key 貼到公開聊天室、社群、GitHub 或截圖裡。
  • 如果懷疑 Key 外洩，立刻到官方 Console 刪掉舊 Key，重新建立新 Key。
  • 不同用途用不同 Key，例如測試用一把、正式用一把。
  • 如果官方支援限制 Key 權限或來源，盡量打開限制。

花費控制

  • 先設定低額度預算、通知或自動加值上限。
  • 剛開始使用便宜模型，確定效果後再換高階模型。
  • 把角色設定、世界觀、歷史訊息控制在真的需要的範圍。
  • 測試群聊時降低角色數和回合數。

11. 常見錯誤怎麼看

看到的狀況 可能原因 可以怎麼做
Invalid API key Key 貼錯、少貼一段、貼到別家的 Key。 回官方 Console 重新複製，確認服務商選對。
Quota exceeded 或 Rate limit 免費額度、每日次數或每分鐘限制用完。 等限制重置、降低聊天頻率、升級付費或換模型。
Billing required 帳號沒有可用點數或尚未設定付款。 到 Billing 頁確認免費額度、預付點數或付款方式。
回答太貴或用量變高 角色設定太長、保留太多歷史、輸出太長或群聊輪數太多。 縮短設定、降低 max response tokens、減少保留訊息和群聊回合。

12. 給完全新手的推薦路線

  1. 先申請 Gemini API Key，因為它最適合用免費層級測試基本連線。
  2. 在 DesktopST 裡貼上 Key，選一個輕量模型，送出一句短短的測試訊息。
  3. 成功後再試 OpenAI、Grok 或 Claude。需要付款的服務先小額加值，不要一開始開高上限。
  4. 每次新增角色卡或群聊功能時，都先用短對話測試，確認用量正常再長聊。
